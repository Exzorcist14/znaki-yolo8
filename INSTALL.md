# Распознавание дорожных знаков — инструкция по запуску

Мобильное Android-приложение, распознающее дорожные знаки в реальном времени с камеры
и озвучивающее их голосом.

В составе сборки:
- модель **YOLOv8 INT8** (TFLite), 55 классов, обучена на ~40 000 фотографий;
- камера — через **CameraX**, оверлей с зелёными рамками поверх превью;
- озвучка — системный **Text-to-Speech** Android (оффлайн).

## Что требуется

- Android-смартфон с **Android 7.0 (Nougat) и новее**.
- Включённая разрешённая установка приложений из неизвестных источников.
- Для голосовой озвучки на русском — установленный **Google TTS** с русским
  голосовым пакетом (на большинстве современных Android он уже есть).

## Установка APK

1. Скопируйте файл `dps_znak_v1.apk` (или `app-debug.apk`) на телефон любым удобным
   способом: USB-кабель, мессенджер, облачное хранилище, e-mail.
2. Откройте файл на телефоне через файловый менеджер или из уведомления о загрузке.
3. Если система спросит «Разрешить установку приложений из этого источника?» —
   разрешите и подтвердите установку.
4. После установки в списке приложений появится иконка **dps_zn**. Запустите.
5. При первом запуске разрешите доступ к камере.

> Если ничего не происходит при тапе по APK — значит ОС блокирует установку.
> На современных Android: **Настройки → Приложения → (выбрать ваш файловый
> менеджер или браузер) → Разрешения → Установка неизвестных приложений →
> Разрешить**. После этого повторите шаг 2.

## Что вы увидите

- Превью с задней камеры на весь экран.
- При обнаружении знака — зелёная рамка вокруг него и подпись на русском
  (например: «Главная дорога 87%»).
- Сверху: заголовок, значок громкости (тап = быстрый mute/unmute) и шестерёнка
  настроек.

## Настройки (шестерёнка справа сверху)

- **Озвучка распознанных знаков** — включает/выключает голос. Один и тот же знак
  не повторяется чаще раза в 6 секунд, чтобы не спамил.
- **Статус голоса** — показывает, готов ли движок:
  - зелёный «Голос: русский» — всё работает;
  - жёлтый «Русский голос не установлен» — нажмите «Настройки голоса» и
    установите русский пакет в системе;
  - красный «TTS-движок не найден» — поставьте Google TTS из Google Play.
- **Тест** — проговаривает тестовую фразу, чтобы сразу проверить, слышно ли голос.
- **Порог уверенности** — рекомендуется **30–50 %**. Ниже — больше ложных
  срабатываний; выше — модель пропускает дальние знаки.

## Как проверять (важные нюансы)

1. **Расстояние и размер знака в кадре**. Лучше всего распознаются знаки, которые
   занимают примерно **1/4–1/3 кадра по ширине**. Если поднести знак вплотную,
   так что он почти заполняет экран, точность может упасть — таких больших
   примеров в обучающем наборе было меньше. Для теста подносите телефон с
   расстояния 1–3 метра до напечатанного знака или показывайте через монитор
   с дистанции вытянутой руки.
2. **Освещение и фон**. Лучше всего работает при равномерном дневном свете,
   на однородном фоне. Сильные блики, темнота и пёстрый фон закономерно
   ухудшают качество.
3. **Удержание в кадре**. Знак должен попадать в кадр **полностью**, без
   обрезки краёв.
4. **Стабильность**. Распознавание идёт покадрово, поэтому лейбл может на
   доли секунды «мигать» между похожими классами — это нормально для модели
   такого размера.

## Запуск через ADB (опционально, для технического пользователя)

Если в системе настроен Android SDK / `adb`:

```bash
adb devices
adb install -r dps_znak_v1.apk
adb shell monkey -p com.example.dps_zn -c android.intent.category.LAUNCHER 1
```

## Известные ограничения текущей модели

Это **прототип / proof-of-concept** для дипломной работы, не production-решение.
Модель обучена на ограниченном наборе из 55 классов, и часть визуально похожих
знаков может путаться (например, треугольные предупреждающие знаки между собой).
Для каждого класса целенаправленно протестируйте на примерах из обучающей
выборки — там точность ожидаемо выше, чем на «новых» уличных фотографиях.

## Контакты

При проблемах с установкой / запуском — приложите снимок экрана с ошибкой
и название модели телефона.
